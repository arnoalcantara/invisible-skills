# FICHA — Passo a passo / fluxo

**Família:** D — Processo, sequência e mudança · **ID:** `passo-a-passo`

**Função didática.** Mostrar uma sequência **ordenada** de etapas, deixando claro que a ordem importa (passo 2 depende do 1). O trabalho na cabeça do aluno: internalizar um procedimento ou um encadeamento, vendo o todo e a posição de cada parte. A direção (setas, numeração) é o que separa este tipo de uma lista comum.

**Posição no arco.** Desenvolver. Costuma vir depois de o "porquê" do processo ter sido estabelecido.

**Quando usar.** Há uma sequência de etapas com ordem obrigatória ou recomendada (um método, um procedimento, um algoritmo, uma receita conceitual). De 3 a 6 passos.

**Quando NÃO usar.** Itens sem ordem entre si → não é fluxo, é lista/taxonomia. Sequência de *eventos no tempo* histórico → [linha-do-tempo](linha-do-tempo.md). Caminhos condicionais (se/então) → [arvore-de-decisao](arvore-de-decisao.md). Processo que se repete sem fim → [ciclo-loop](ciclo-loop.md).

**Anatomia (slots).**
- *Título-asserção* (obrigatório) — afirma o que o processo conquista, não "As etapas do processo".
- *Passos numerados* (obrigatório) — 3 a 6, cada um com rótulo curto + uma linha de descrição.
- *Conectores de direção* (obrigatório) — setas ou numeração que tornam a ordem inequívoca.
- *Destaque do passo crítico* (opcional) — se um passo é o gargalo ou o mais errado na prática.

**Regras de carga.** 3 a 6 passos (mais que 6 → quebre em dois slides ou agrupe). Cada passo: um rótulo (2–4 palavras) + no máximo uma frase. Direção visualmente única (todas as setas no mesmo sentido). Sem parágrafos dentro do passo.

**Build.** Fortemente recomendado: revele passo a passo, no ritmo da explicação. É o tipo onde a revelação progressiva (lei 8) mais rende — o aluno não corre à frente nem se perde.

**Decisão visual (roteamento).** **Modo 0** — caixas e setas em HTML/CSS/SVG, com rótulos precisos. Nunca um "fluxograma" gerado por modelo de imagem (erra rótulo e seta). Geração só se cada etapa precisar de uma *ilustração conceitual* ao lado, e aí cada ilustração vai pelo caminho próprio.

**Conexão posicional.** *Pede antes:* o objetivo/porquê do processo e o vocabulário das etapas pré-treinado. *Aponta para:* um exemplo resolvido que aplica o processo, ou um ponto de processamento (o aluno executa um passo).

**Erro comum.** Listar etapas sem direção visível (vira lista, perde o "ordem importa"). Passos demais num slide só. Descrição longa em cada passo (viola carga). Título-tópico.

**Por dial de fidelidade.** *Transcrição:* os passos saem da sequência que o professor descreveu, na ordem dele. Não acrescente um passo que ele não mencionou para "completar" o método. *Esboço:* você estrutura o processo, podendo pesquisar a sequência canônica, marcada para aprovação.
