# Tipologia — as 8 famílias de slide

> **Propósito.** O catálogo dos tipos de slide, organizado por **função didática** (o trabalho que o slide faz na cabeça do aluno), não por aparência. Na Passada 1 você classifica cada trecho pela *intenção* → escolhe a família → escolhe o tipo. Na Passada 2 você abre a **ficha** do tipo (em [fichas/](fichas/)) e instancia.
> **Lugar na cadeia:** terceira camada (filosofia → arco → **tipologia** → produção visual).

---

## Como usar

1. Pergunte: *que trabalho este slide precisa fazer na cabeça do aluno?* (apresentar uma ideia? contrastar? mostrar um processo? fazer o aluno agir?)
2. Isso aponta a **família** (A–H abaixo).
3. Dentro da família, escolha o **tipo** pela situação concreta.
4. Abra a **ficha** do tipo (link na tabela). A ficha tem a anatomia, as regras de carga (números), o build, a decisão visual e o erro comum.

**Notas de implementação:**
- **Primitivos vs. compostos.** Prefira primitivos (um trabalho só). Infográfico, dashboard e jornada anotada são *compostos* — só componha quando a carga justificar.
- **Revelação progressiva é ortogonal.** Não é um tipo; é um *atributo* (a lei 8) aplicável a quase todos os tipos. A ficha de cada tipo diz se e como admite build.
- **Tipo-padrão.** Quando em dúvida sobre como apresentar uma ideia, o default é **asserção-evidência** (família B) — a coluna vertebral da skill.
- **Profundidade das fichas.** Os tipos de alta frequência têm ficha completa (gabarito cheio); os demais têm ficha enxuta (campos essenciais), suficiente para gerar com consistência. Todas seguem o mesmo gabarito.

## A — Orientação e estrutura
Slides que dizem ao aluno **onde ele está** no argumento. Realizam a lei 13 (espinha narrativa).

| Tipo | Ficha | Quando usar |
|---|---|---|
| Abertura / título | [abertura-titulo](fichas/abertura-titulo.md) | Primeiro slide. Fisga e nomeia a aula. |
| Objetivos | [objetivos](fichas/objetivos.md) | Logo após a abertura. O que o aluno saberá fazer ao fim. |
| Roteiro / agenda | [roteiro-agenda](fichas/roteiro-agenda.md) | Mapa da aula em aulas longas ou com muitos blocos. |
| Divisor de seção | [divisor-secao](fichas/divisor-secao.md) | Marca a virada para um novo bloco. Respiro. |
| Você está aqui | [voce-esta-aqui](fichas/voce-esta-aqui.md) | Retoma o mapa mostrando o progresso, em aulas longas. |
| Ponte / transição | [ponte-transicao](fichas/ponte-transicao.md) | Conecta o que veio ao que vem; fecha um gancho, abre outro. |
| Ativação de conhecimento prévio | [ativacao-conhecimento-previo](fichas/ativacao-conhecimento-previo.md) | Antes de conteúdo novo; puxa a âncora do que o aluno já sabe. |

## B — Apresentar uma ideia
O coração expositivo. Uma ideia entra na cabeça do aluno.

| Tipo | Ficha | Quando usar |
|---|---|---|
| Asserção + evidência **(tipo-padrão)** | [asseracao-evidencia](fichas/asseracao-evidencia.md) | Default para apresentar qualquer ideia: título afirma, visual prova. |
| Declaração-soco | [declaracao-soco](fichas/declaracao-soco.md) | Uma afirmação forte sozinha na tela, para fixar ou chocar. |
| Definição | [definicao](fichas/definicao.md) | Introduzir um termo com precisão. Vale dobrado em conteúdo técnico/doutrinário. |
| Citação / autoridade | [citacao-autoridade](fichas/citacao-autoridade.md) | A voz de uma fonte confiável sustenta o ponto. |

| Tipo | Ficha | Quando usar |
|---|---|---|

## E — Dados e quantidade
Trazem números à tela — com **uma** mensagem, nunca um despejo de dados.

| Tipo | Ficha | Quando usar |
|---|---|---|
| Gráfico (uma mensagem) | [grafico](fichas/grafico.md) | Um dado que precisa estar correto e tem um ponto a destacar. |
| Número grande único | [numero-grande](fichas/numero-grande.md) | Uma métrica de impacto sozinha na tela. |
| Destaque sobre dado | [destaque-sobre-dado](fichas/destaque-sobre-dado.md) | Apontar o que olhar dentro de um dado/gráfico. |
| Infográfico | [infografico](fichas/infografico.md) | Composição de dados+arte. Composto — com parcimônia. |

| Tipo | Ficha | Quando usar |
|---|---|---|

## G — Processamento ativo
Fazem o aluno **agir**. Realizam a lei 12.

| Tipo | Ficha | Quando usar |
|---|---|---|
| Pergunta / provocação | [pergunta-provocacao](fichas/pergunta-provocacao.md) | Abrir uma lacuna; fazer o aluno querer a resposta. |
| Predição | [predicao](fichas/predicao.md) | Pedir um palpite antes de revelar — fixa pela surpresa. |
| Enquete / mão pra cima | [enquete](fichas/enquete.md) | Medir a sala e engajar com uma escolha rápida. |
| Pense-converse-compartilhe | [pense-converse-compartilhe](fichas/pense-converse-compartilhe.md) | Atividade em pares/grupo no meio da aula. |
| Preencher lacuna / revelar | [preencher-lacuna](fichas/preencher-lacuna.md) | Esconder a resposta, deixar o aluno completar, revelar. |
| Problema para resolver | [problema-para-resolver](fichas/problema-para-resolver.md) | Pôr um problema na tela para a sala atacar. |
| Verificação de entendimento / quiz | [verificacao-entendimento](fichas/verificacao-entendimento.md) | Checar se o ponto pegou antes de avançar. |
