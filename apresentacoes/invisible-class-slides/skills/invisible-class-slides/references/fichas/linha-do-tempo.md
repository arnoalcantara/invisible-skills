# FICHA — Linha do tempo

**Família:** D — Processo, sequência e mudança · **ID:** `linha-do-tempo`

**Função didática.** Posicionar eventos numa **cronologia**, tornando visível a ordem, o intervalo entre eles e (quando útil) a aceleração/desaceleração. O trabalho na cabeça do aluno: ancorar fatos no tempo e ver a forma do período (o que veio antes, o que se acelerou, o que demorou).

**Posição no arco.** Desenvolver. Boa para dar a estrutura de um período antes de mergulhar nos detalhes de cada momento (pré-treino), ou para consolidar uma sequência já discutida.

**Quando usar.** Eventos datados ou ordenados no tempo (história, evolução de uma ideia, fases de um projeto). De 3 a 7 marcos.

**Quando NÃO usar.** Etapas de um procedimento (sem datas, ordem lógica) → [passo-a-passo](passo-a-passo.md). Um único antes→depois → [antes-depois](antes-depois.md). Mais de ~7 marcos densos → quebre por era/fase em slides sucessivos.

**Anatomia (slots).**
- *Título-asserção* (obrigatório) — afirma o que a cronologia revela ("A reforma levou um século para se consolidar"), não "Linha do tempo".
- *Eixo do tempo* (obrigatório) — horizontal em geral; com noção de escala se os intervalos importam.
- *Marcos* (obrigatório) — 3 a 7, cada um com data/rótulo + um fato curto.
- *Destaque do marco-chave* (opcional) — o ponto de virada.

**Regras de carga.** 3 a 7 marcos. Cada marco: data + rótulo + no máximo uma frase. Se os intervalos importam, o espaçamento no eixo deve refleti-los (senão, distorce a leitura). Um eixo, uma direção.

**Build.** Recomendado: revele marco a marco, da esquerda para a direita, acompanhando a narrativa. Útil para construir a tensão até o ponto de virada.

**Decisão visual (roteamento).** **Modo 0** — eixo, pontos e rótulos em HTML/CSS/SVG. Se cada marco pede uma miniatura (um retrato, uma obra), Modo 1 (stock) por marco. Não gere a linha do tempo inteira como imagem (datas e posições erram).

**Conexão posicional.** *Pede antes:* o contexto do período; os atores/conceitos já nomeados. *Aponta para:* a análise de um marco específico, uma causa-efeito entre eventos, ou uma síntese do período.

**Erro comum.** Intervalos visualmente iguais quando os reais são muito diferentes (engana a percepção). Marcos demais (vira tabela deitada). Título-tópico. Texto longo em cada marco.

**Por dial de fidelidade.** *Transcrição:* os marcos e datas saem do que o professor citou; não acrescente eventos "que faltavam" sem marcar como adição. Confira datas — erro factual é grave. *Esboço:* você seleciona os marcos relevantes, podendo pesquisar, marcados para aprovação e conferência de datas.
